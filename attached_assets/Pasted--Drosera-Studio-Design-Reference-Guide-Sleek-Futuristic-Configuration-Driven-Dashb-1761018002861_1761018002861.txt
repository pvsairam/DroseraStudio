## üß≠ 1. Core Design Principles
- **Clarity over complexity:** Display only meaningful data at a glance.  
- **Motion as feedback:** Use subtle transitions, liquid animations, and hover responses.  
- **Futuristic minimalism:** Flat surfaces, blended gradients, and intelligent light use.  
- **Configuration-first philosophy:** All menus, pages, and components dynamically generated.  
- **Typography rhythm:** Wide spacing, geometric font, clear hierarchy.  
- **Human-centric feel:** Every click, scroll, or hover should ‚Äúfeel alive.‚Äù  
- **Performance & accessibility:** WCAG AA compliant, reduced-motion friendly, PWA optimized.

## üß± 3. Design System Tokens

| Element | Specification |
|----------|----------------|
| **Primary Accent** | `#00A3FF` (Electric Blue) |
| **Secondary Accent** | `#FF8A00` (Alert Orange) |
| **Success** | `#2ECC71` |
| **Warning** | `#F1C40F` |
| **Error** | `#E74C3C` |
| **Dark Background** | `#121212` |
| **Light Background** | `#FAFAFA` |

**Typography**
- Font Family: `Inter`, fallback `system-ui`  
- Headings: 24‚Äì32 px, weight 600  
- Body: 16 px, weight 400  
- Captions: 14 px, spacing 0.5 px  

**Layout Grid**
- 12-column responsive grid  
- 8 px base spacing unit  
- 12 px border-radius  
- 0 4 16 px rgba(0, 0, 0, 0.1) for standard shadows  

---

## üîÑ 4. Motion & Interaction
| Motion Type | Description |
|--------------|--------------|
| **Page Transition** | Fade + slide up 20 px, 0.3 s spring ease |
| **Card Hover** | Scale 1.02 + shadow deepen |
| **Chart Load** | Staggered opacity rise (100 ms intervals) |
| **Menu Expand** | Accordion with friction spring |
| **Live Feed Ticker** | Horizontal auto-scroll, pause on hover |

All animations must obey `prefers-reduced-motion`.

---

## üß© 5. Layout Structure
**Header:** Time-range selector, user avatar, theme toggle  
**Sidebar:** Dynamic config-driven navigation  
**Main Area:**  
- Top: KPI cards  
- Middle: Charts (event frequency, trap categories)  
- Bottom: Live feed + status tables  

**Admin Panel:**  
Config forms with live preview & JSON export buttons.

---

## ‚ôø 6. Accessibility & Performance
- Keyboard navigation for every control.  
- Visible focus ring with high contrast.  
- `aria-live="polite"` for trap event feed.  
- Lazy load charts & heavy graphs.  
- Maintain LCP < 2.5 s, CLS < 0.1.  

---

## üö´ 7. Avoid
- Neon gradients, 3D parallax, or ‚ÄúAI-template‚Äù styles.  
- Overcrowded tables or rainbow palettes.  
- Long transition delays or unnecessary complexity.  

---

## üß† 8. Goal Recap
The **Drosera Dashboard** must:
- Feel human, fluid, and intuitive.  
- Offer the precision of **Stripe**, the fluidity of **Linear**, and the modern minimalism of **Vercel**.  
- Be entirely **configuration-driven**, **visually timeless**, and **technically scalable**.  

Build it like a digital command center ‚Äî silent, responsive, and beautiful.
